This is the test plugin that allows work with bets

== Description ==

Backend

Написать собственный плагин для Wordpress.

Плагин должен быть совместим с Wordpress 4.7 и выше. Плагин может быть must-use.

Функционал плагина:

Создать произвольный тип постов «Ставки» (bets).
Добавить таксономию «Тип ставки» для типа постов «Ставки» со следующими терминами:
ординар
экспресс
Добавить таксономию «Статус ставки» для типа постов «Ставки» со следующими терминами:
выигрыш
проигрыш
возврат
активная
 Добавить две произвольные роли:
«Каппер» - со стандартными правами подписчика и возможностью добавлять/редактировать свои ставки в типе постов «Ставки».
«Модератор» - с правами добавлять/редактировать любые ставки в типе постов «Ставки».

Требования:
Все начальные термины таксономий должны быть добавлены программно.
Все текстовые значения/лейблы, должны быть подготовлены для переводов.

Frontend

Создать простую тему (дизайн не имеет значения), в которой будут следующие страницы (помимо стандартных):

1. Произвольный шаблон страницы (page template), с формой добавления ставки. При отправки формы создается новый пост (bets), отправка должна осуществляться через ajax. Форма должна содержать следующие поля и доступна только авторизованным:

Заголовок (text)
Описание (textarea)
Тип ставки (select)
Кнопка отправить

2. На созданных страницах, назовем их одиночной ставкой (single-bets.php) должно находиться:

Описание, тип ставки, статус. 
Поле input, в котором можно ввести значения от 100 до 1000.  
Кнопка «Ставка пройдет!».  По клику на нее записываются данные из input поля в post meta «_bet_vote». Запись должна происходить через ajax. После чего добавляется cookie пользователю и кнопка становится неактивной для этого пользователя пока существуют куки.

Требования:
Все текстовые значения/лейблы, должны быть подготовлены для переводов.
Должна быть валидация всех входящих данных.
Тема должна содержать два сайдбара (боковая колонка и подвал).
Тема должна поддерживать миниатюры записей.

Разрешено использование любых средств frontend разработки. Все что не указано, остается на усмотрение кандидата.
